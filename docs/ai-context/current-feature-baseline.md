# Current Feature Baseline and Parity Gap

Last audited: 2026-06-06

This document is for future AI agents working on `hswolff.com`. Treat it as the preservation baseline before redesigns. A redesign can change layout, information architecture, and visual style, but should intentionally preserve or replace the capabilities listed here.

## Current Site Purpose

`hswolff.com` is Harry Wolff's personal website and blog. It presents Harry as a software engineering leader, JavaScript/React educator, writer, YouTube creator, and builder of side projects. The site also hosts a standalone Dough Maker product/privacy page.

## Current Top-Level Surfaces

- Home: `/`
  - Introductory hero with profile image, current professional positioning, links to MongoDB, YouTube, and the blog.
  - Featured posts section exists if any blog post has `featured: true`.
  - Recent posts section shows the four newest posts.
  - Link to all blog posts.
- Blog index: `/blog/`
  - Paginated list of blog posts.
  - Uses `SITE.postPerPage`, currently 10 posts per page.
  - Pagination pages are routed through `/blog/[slug].astro`; numeric slugs act as page numbers.
- Blog detail pages: `/blog/{postSlug}/`
  - Render Markdown content from `src/content/blog`.
  - Show title, date, category, reading time, and tags.
  - Include a "Go back" button using browser history.
  - Include Disqus comments.
  - Use article Open Graph metadata.
- Category archives: `/blog/category/{category}/`
  - Categories are currently strict: `code`, `personal`, and `career`.
  - Each category archive lists non-draft posts in that category.
- Tag archive index: `/blog/tags/`
  - Lists all unique tags.
  - The nav link is currently commented out, so this page exists but is not visible in the main nav.
- Tag archives: `/blog/tags/{tag}/`
  - Lists non-draft posts for a tag.
- Search: `/search`
  - Client-side React search powered by Fuse.js.
  - Searches post title and description.
  - Syncs the query into the `?q=` URL parameter.
  - Autofocuses the search input on load.
- Projects: `/projects/`
  - Static Markdown page for YouTube, open source, and unmaintained projects.
- Uses: `/uses/`
  - Static Markdown page listing tools, desk setup, recording setup, and a YouTube iframe.
  - Content says it was last updated on February 14, 2021.
- About: `/about/`
  - Static Markdown biography page with profile image.
- Contact: `/about/contact/`
  - Static contact page with a React contact form.
  - The form is configured as a Netlify form but the site adapter is Vercel; verify behavior before relying on it.
- Talks: `/about/talks/`
  - Lists blog posts tagged `talk`.
  - This page exists but is not visible in the main nav.
- Newsletter: `/newsletter/`
  - Static Markdown newsletter pitch page with a YouTube iframe.
  - This page exists but is not visible in the main nav.
- Dough Maker: `/dough-maker`
  - Standalone Astro page outside the shared site layout.
  - Contains product positioning, feature cards, FAQ, app store CTA, support email, privacy policy, SoftwareApplication JSON-LD, and page-specific styling.
- RSS feed: `/rss.xml`
  - Generated with `@astrojs/rss`.
  - Includes full sanitized Markdown-rendered content.
  - Current implementation collects all blog posts; unlike search and detail pages, it does not filter drafts.
- Robots: `/robots.txt`
  - Allows all user agents except a `Googlebot` disallow for `/nogooglebot/`.
  - Includes the generated sitemap URL.
- Sitemap: `/sitemap-index.xml`
  - Generated by `@astrojs/sitemap`.
- 404: `/404`
  - Custom 404 page exists.
- Dynamic Open Graph image route: `/[ogTitle].svg`
  - Route and generator exist, but `getStaticPaths()` currently returns `[]`, so static OG generation is effectively disabled unless revisited.

## Blog Content Baseline

- Current post count: 65 Markdown posts under `src/content/blog`.
- Date range: 2010-12-28 through 2020-10-27.
- Content is organized by year directories.
- Blog schema is strict and lives in `src/content/_schemas.ts`.
- Required frontmatter:
  - `title`
  - `date`
  - `category`
  - `postSlug`
- Optional frontmatter:
  - `updated`
  - `tags`
  - `draft`
  - `featured`
  - `ogImage`
  - `description`
  - `canonicalURL`
- Current audit found no posts with `draft: true` and no posts with `featured: true`.
- Slugs are generated from frontmatter with `src/utils/slugify.ts`, not from filenames.
- `description` is used by search data when present; the blog detail layout uses a generated excerpt from the rendered Markdown tree for page description.

## Current Accessibility Features

- Skip-to-content link is present.
- Header nav button has `aria-label`, `aria-expanded`, and `aria-controls`.
- Search icon in search input is `aria-hidden`.
- Date/category icons are `aria-hidden`.
- Blog date has screen-reader-only "Posted on:" text.
- Pagination nav has `aria-label="Pagination"`.
- Theme toggle uses `aria-label` and `aria-live`.
- Most visible images have alt text.

Accessibility caveats to preserve awareness of during redesign:

- Dough Maker uses emoji in headings and a separate custom design system.
- Some historical Markdown content may include legacy iframes/images that need audit.
- Contact form error handling currently uses `alert(error)`.
- The search component assumes `inputRef.current` exists after load.

## Personal Website and Blog Feature Checklist

This checklist combines current best-practice themes from Google Search Central, W3C/WCAG guidance, MDN metadata guidance, and contemporary 2026 personal-site/portfolio guidance. Sources are listed at the bottom.

Must-have baseline for a strong personal website:

- Clear homepage positioning: who Harry is, what he does, what the visitor should do next.
- Fresh About page with current role, credibility, personality, and a clear contact path.
- Strong project/work page with selected projects, outcomes, role, links, screenshots, and recency.
- Blog/writing archive with clear post pages, categories/tags, search, RSS, and internal links.
- Contact path: email, form, social DM, booking link, or clear collaboration CTA.
- Social proof: talks, podcasts, videos, companies, testimonials, metrics, or featured work.
- Newsletter or subscription path if audience growth is a goal.
- Uses/resources page if it serves recurring audience questions.
- Privacy/legal pages when collecting forms, analytics, newsletter signups, or product data.
- Accessible, responsive, keyboard-operable UI.
- Technical SEO: unique titles/descriptions, canonical URLs, sitemap, robots, crawlable links, structured data where useful, good image alt text, descriptive URLs.
- Sharing metadata: Open Graph and Twitter cards with useful images.
- Performance-oriented static pages, optimized images, minimal client JavaScript.
- Maintenance signals: visible recency, updated project information, current role, and no dead CTAs.
- AI-agent/LLM discoverability: clear entity info, structured biographical facts, explicit project descriptions, contact data, and crawlable long-form content.

Nice-to-have features for a top-tier personal website/blog:

- Dedicated "Start here" or "Best of" writing page.
- Featured writing collections by topic.
- Speaking/media page linked in main navigation.
- Case-study project pages instead of only outbound links.
- Blog post table of contents where posts are long.
- Related posts / next-previous post navigation.
- Newsletter archive or subscription embed that actually signs users up.
- `/now` page or changelog for current work.
- Colophon/site source page.
- JSON-LD Person schema on the personal site and Article schema on posts.
- Webmentions or modern comments alternative, if comments matter.
- Analytics that match the current deployment provider and privacy posture.
- Image optimization for profile and post images.
- Explicit redirect/legacy URL strategy for old posts during redesign.

## Parity Gap

High-priority gaps before or during redesign:

- Content recency gap: the newest audited blog post is dated 2020-10-27, and `/uses/` says it was last updated on 2021-02-14.
- Positioning gap: the homepage still says Harry is a software engineering manager currently working at MongoDB, while `/about/` says Lead Engineer. Verify and unify current positioning before redesign.
- Project proof gap: `/projects/` is mostly narrative/outbound links. It lacks selected case studies, screenshots, outcomes, current app work, and stronger proof-of-work framing.
- Contact gap: `/about/contact/` exists but is not in the main nav. The contact form is Netlify-style while the app deploys with the Vercel adapter, so form delivery may not work.
- Newsletter gap: `/newsletter/` exists but is not in the nav and has no visible signup/embed beyond an explanatory page and YouTube iframe.
- Speaking/media gap: `/about/talks/` exists but is not in the nav. It could become a first-class social-proof page.
- Featured content gap: the homepage has a featured-post section, but no posts are currently marked featured.
- OG image gap: custom OG generation code exists, but static paths are disabled.
- Draft/feed consistency gap: search and detail pages filter drafts, but RSS currently does not.

Medium-priority gaps:

- No dedicated "best of" page for new readers.
- No related-post or previous/next navigation on blog details.
- No visible tag browsing in the main nav despite tag pages existing.
- No current Person JSON-LD for Harry on the personal site.
- Blog posts do not appear to emit Article JSON-LD beyond `og:type=article`.
- No explicit privacy page for analytics/contact/newsletter on the personal site, separate from Dough Maker.
- Disqus adds third-party comments but may not fit a modern privacy/performance posture.
- Social links include Facebook and Twitter/X; verify which channels should remain active.
- `SITE.SOCIALS` contains many inactive placeholder links copied from Astro Paper.

Lower-priority gaps:

- `LOCALE` is set to `en-EN`; `en-US` may be more appropriate for a New York/US personal site.
- Profile image dimensions use `LOGO_IMAGE.width/height` even when the logo image feature is disabled.
- Google Analytics ID is the older `UA-12625863-1` format; verify current analytics behavior.
- The nav only has one responsive breakpoint (`sm: 640px`), which may constrain more ambitious responsive layouts.

## Redesign Preservation Checklist

Any redesign should explicitly account for:

- Keep existing public URLs working or add redirects.
- Preserve the 65-post blog archive and `postSlug` URL behavior.
- Preserve RSS, sitemap, robots, search, category archives, and tag archives unless intentionally replaced.
- Preserve light/dark support or intentionally remove it with user approval.
- Preserve Dough Maker page content and metadata.
- Preserve SEO metadata, canonical URLs, Open Graph/Twitter cards, and article metadata.
- Preserve accessible keyboard navigation, skip link, focus states, and mobile nav semantics.
- Decide what to do with contact, newsletter, comments, and analytics rather than leaving them ambiguous.
- Re-run build and inspect generated routes after major changes.
